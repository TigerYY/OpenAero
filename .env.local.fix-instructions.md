# 修复 DATABASE_URL 中的密码编码

你的密码包含特殊字符：`Apollo202%251419$`

在 PostgreSQL 连接字符串中，特殊字符需要正确编码：
- `%` 需要编码为 `%25`
- `$` 需要编码为 `%24`
- `#` 需要编码为 `%23`
- 等等

当前你的 .env.local 中：
DATABASE_URL="postgresql://postgres.cardynuoazvaytvinxvm:Apollo202%251419$@aws-1-us-east-2.pooler.supabase.com:5432/postgres"

问题：密码中的 `$` 字符没有编码。

## 修复方法

将 .env.local 中的 DATABASE_URL 和 DIRECT_URL 更新为：

DATABASE_URL="postgresql://postgres.cardynuoazvaytvinxvm:Apollo202%25251419%24@aws-1-us-east-2.pooler.supabase.com:5432/postgres"
DIRECT_URL="postgresql://postgres.cardynuoazvaytvinxvm:Apollo202%25251419%24@aws-1-us-east-2.pooler.supabase.com:5432/postgres"

注意：
- `%25` 保持不变（已经是编码后的 %）
- `$` 编码为 `%24`

或者，更安全的方法是使用 Supabase Dashboard 中显示的完整连接字符串（包含正确编码的密码）。
