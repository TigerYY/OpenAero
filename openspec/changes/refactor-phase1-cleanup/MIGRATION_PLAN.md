# 路由迁移计划

## 分析结果

### 重复路由情况

1. **认证路由** (login, register, forgot-password)
   - 非国际化版本：仅重定向页面
   - 国际化版本：完整功能页面
   - **策略**: 删除非国际化版本，依赖中间件处理重定向

2. **创作者路由** (creators/*)
   - 非国际化版本：5个页面（部分为占位页面）
   - 国际化版本：5个页面（完整功能）
   - **策略**: 比较内容，保留国际化版本，删除或迁移非国际化版本

3. **管理路由** (admin/*)
   - 非国际化版本：10个页面（完整功能）
   - 国际化版本：1个页面（monitoring）
   - **策略**: 迁移所有admin页面到国际化目录

## 迁移步骤

### 阶段1: 认证路由清理
1. 删除 `src/app/login/page.tsx`（重定向页面）
2. 删除 `src/app/register/page.tsx`（如果存在）
3. 删除 `src/app/forgot-password/page.tsx`（如果存在）
4. 验证中间件正确处理重定向

### 阶段2: 创作者路由清理
1. 比较 `src/app/creators/*` 和 `src/app/[locale]/creators/*` 的内容
2. 如果内容相同，删除非国际化版本
3. 如果内容不同，迁移到国际化版本
4. 更新所有引用

### 阶段3: 管理路由迁移
1. 迁移 `src/app/admin/*` 到 `src/app/[locale]/admin/*`
2. 更新所有admin路由引用
3. 验证功能正常

## 注意事项

- 保持向后兼容（如果需要）
- 更新所有硬编码路由引用
- 测试所有迁移的路由功能
